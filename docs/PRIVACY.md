# Персональные данные (ПДн) — QuickSave Bot

Краткая политика для пользователей и операторов. Юридическое оформление при необходимости согласуйте с юристом под вашу юрисдикцию.

## Какие данные обрабатываются

| Данные | Назначение |
|--------|------------|
| `telegram_user_id` | Идентификация пользователя в боте, привязка сохранений к аккаунту |
| `username` (если указан в Telegram) | Удобство в логах и отладке; может отсутствовать |
| Текст сообщений (заметки, ссылки) | Основной функционал — хранение пользовательского контента |
| `telegram_update_id` | Идемпотентность: защита от повторной обработки одного апдейта |

Файл SQLite (`DATABASE_PATH`, по умолчанию `./data/quicksave.db`) хранится на диске сервера/контейнера. Резервное копирование — ответственность оператора (см. [DEPLOY.md](DEPLOY.md)).

## Логи

Структурированные логи (Pino) в stdout: метаданные запросов, ошибки, счётчики. **Содержимое пользовательских сообщений в логи не пишется** при штатной работе. Уровень детализации — `LOG_LEVEL` (по умолчанию `info`).

## Срок хранения

Специального автоудаления в MVP нет: данные хранятся до удаления оператором или удаления файла БД. При необходимости можно добавить команду удаления данных пользователя или job очистки — отдельная задача.

## Права пользователя

Пользователь может:

- прекратить использование бота (новые данные не создаются при отсутствии обращений к боту);
- запросить у оператора проекта удаление своих данных из БД (оператор удаляет строки по `telegram_user_id` вручную или через будущий инструмент).

## Передача третьим лицам

Данные не продаются и не передаются третьим лицам, кроме инфраструктуры, на которой крутится бот (хостинг), в объёме, необходимом для работы сервиса. Токен бота и доступ к БД — только у операторов согласно [SECRETS.md](SECRETS.md).

## Контакт

Вопросы по данным — к владельцу развёртывания бота (email/канал, указанные в вашем проекте).
